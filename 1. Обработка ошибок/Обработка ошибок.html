Что такое исключения

Исключение (exception) — это событие, которое возникает во время выполнения программы и нарушает её нормальный ход. Когда Python сталкивается с ошибкой, он создаёт объект исключения и прерывает выполнение текущего кода.

Зачем нужна обработка исключений?

  1. Предотвращает аварийное завершение программы при возникновении ошибок.
  2. Позволяет реагировать на ошибки и выполнять альтернативные действия.
  3. Делает код более надёжным и предсказуемым.
  4. Помогает выводить понятные сообщения об ошибках для пользователя.

Пример программы, которая завершится с ошибкой без обработки исключений:

number = int(input("Введите число: "))  # пользователь вводит "abc"
print(f"Вы ввели: {number}")

# ValueError: invalid literal for int() with base 10: 'abc'

Встроенные исключения

Python содержит множество встроенных классов исключений. Каждый из них отвечает за определённый тип ошибки. Все исключения наследуются от базового класса BaseException.

Исключение Описание Пример
ValueError Неверное значение аргумента (тип правильный, значение нет) int("abc")
TypeError Операция применена к объекту неподходящего типа "5" + 5
KeyError Обращение к несуществующему ключу словаря d["key"] при отсутствии "key"
IndexError Обращение к индексу, выходящему за границы последовательности [1, 2, 3][10]
AttributeError Обращение к несуществующему атрибуту объекта "text".sort()
NameError Обращение к неопределённой переменной print(x) без объявления x
ZeroDivisionError Деление на ноль 10 / 0
FileNotFoundError Файл или директория не найдены open("file.txt") при отсутствии файла
ImportError Ошибка при импорте модуля import несуществующий_модуль
StopIteration Итератор исчерпан next(iter([]))
RuntimeError Общая ошибка во время выполнения, не попадающая в другие категории Нарушение глубины рекурсии
OverflowError Результат операции слишком велик для представления math.exp(1000)
MemoryError Недостаточно памяти для выполнения операции Создание очень большого списка
RecursionError Превышена максимальная глубина рекурсии Бесконечная рекурсия
OSError Ошибка операционной системы (файлы, сеть и т.д.) Отказ в доступе к файлу

Примеры возникновения исключений:

int("hello")        # ValueError
"text" + 5          # TypeError
{"a": 1}["b"]       # KeyError
[1, 2, 3][99]       # IndexError
10 / 0              # ZeroDivisionError
print(undefined_var) # NameError

Иерархия исключений

Все исключения наследуются от BaseException. Большинство стандартных исключений наследуются от Exception. Это важно при обработке: если перехватить Exception, то будут пойманы все дочерние исключения.

BaseException
 ├── SystemExit
 ├── KeyboardInterrupt
 └── Exception
      ├── ValueError
      ├── TypeError
      ├── KeyError
      │    └── IndexError (косвенно через LookupError)
      ├── ZeroDivisionError
      └── ... (и многие другие)

try / except / else / finally

Для обработки исключений в Python используется конструкция try / except. Она позволяет «поймать» исключение и выполнить альтернативный код вместо аварийного завершения программы.

try:
    # код, который может вызвать исключение
except ТипИсключения:
    # код, выполняемый при возникновении исключения
else:
    # код, выполняемый если исключений НЕ было
finally:
    # код, выполняемый ВСЕГДА (и при ошибке, и без)

Блоки конструкции

  • try — обязательный блок. Содержит код, который может вызвать исключение.
  • except — обязательный (хотя бы один). Перехватывает указанное исключение и выполняет код обработки ошибки.
  • else — необязательный. Выполняется только если в блоке try исключений не возникло.
  • finally — необязательный. Выполняется всегда, независимо от того, было ли исключение. Используется для освобождения ресурсов.

Простой пример: обработка ввода пользователя

try:
    number = int(input("Введите число: "))
    result = 100 / number
except ValueError:
    print("Ошибка: введите целое число!")
except ZeroDivisionError:
    print("Ошибка: нельзя делить на ноль!")
else:
    print(f"Результат: {result}")
finally:
    print("Программа завершена.")

Получение информации об исключении

С помощью ключевого слова as можно получить объект исключения и вывести его сообщение:

except ТипИсключения as e:
Сохраняет объект исключения в переменную e. Можно использовать str(e) или просто e для получения сообщения об ошибке. 
try:
    data = {"name": "Alice"}
    print(data["age"])
except KeyError as e:
    print(f"Ключ не найден: {e}")
    # Ключ не найден: 'age'

Перехват любого исключения

Можно использовать Exception для перехвата любого стандартного исключения. Это удобно, но следует применять осторожно — лучше указывать конкретные типы исключений.

try:
    result = 10 / 0
except Exception as e:
    print(f"Произошла ошибка: {type(e).__name__} — {e}")
    # Произошла ошибка: ZeroDivisionError — division by zero

Пример с блоком else и finally

def read_file(filename):
    try:
        file = open(filename, "r")
        content = file.read()
    except FileNotFoundError:
        print(f"Файл '{filename}' не найден.")
        content = None
    else:
        print("Файл успешно прочитан.")
        file.close()
    finally:
        print("Операция завершена.")
    return content

read_file("data.txt")

Несколько блоков except

Один блок try может содержать несколько блоков except для обработки различных типов исключений по-разному.

def safe_divide(a, b):
    try:
        result = a / b
    except TypeError:
        print("Ошибка: оба аргумента должны быть числами.")
    except ZeroDivisionError:
        print("Ошибка: делитель не может быть равен нулю.")
    else:
        return result

print(safe_divide(10, 2))    # 5.0
print(safe_divide(10, 0))    # Ошибка: делитель не может быть равен нулю.
print(safe_divide("10", 2))  # Ошибка: оба аргумента должны быть числами.

Перехват нескольких исключений в одном блоке

Если для нескольких типов исключений нужна одинаковая обработка, их можно объединить в кортеж:

try:
    value = int(input("Введите число: "))
    result = 100 / value
except (ValueError, ZeroDivisionError) as e:
    print(f"Ошибка ввода: {e}")

Порядок блоков except важен

Блоки except проверяются сверху вниз. Следует размещать более конкретные исключения выше, а более общие — ниже. Иначе общий обработчик перехватит исключение до того, как до него дойдёт конкретный.

try:
    result = int("abc") / 0
except ValueError:
    print("Ошибка значения")      # более конкретное — выше
except ZeroDivisionError:
    print("Деление на ноль")      # более конкретное — выше
except Exception:
    print("Общая ошибка")         # более общее — ниже

raise — генерация исключений

Иногда нужно самостоятельно вызвать исключение — например, при проверке входных данных в функции. Для этого используется ключевое слово raise.

raise ТипИсключения("сообщение")
Генерирует исключение указанного типа с заданным сообщением.

Пример: валидация данных

def set_age(age):
    if not isinstance(age, int):
        raise TypeError("Возраст должен быть целым числом.")
    if age < 0 or age > 150:
        raise ValueError("Возраст должен быть в диапазоне от 0 до 150.")
    return age

try:
    set_age(-5)
except ValueError as e:
    print(f"Ошибка: {e}")
# Ошибка: Возраст должен быть в диапазоне от 0 до 150.

Повторная генерация исключения (re-raise)

Внутри блока except можно использовать raise без аргументов, чтобы пробросить текущее исключение дальше — после его частичной обработки:

def process_data(data):
    try:
        result = 100 / data
    except ZeroDivisionError:
        print("Логируем ошибку деления на ноль...")
        raise  # пробрасываем исключение дальше

try:
    process_data(0)
except ZeroDivisionError:
    print("Исключение поймано в вызывающем коде.")

raise ... from ...

Можно указать причину (цепочку исключений) с помощью raise ... from ...:

try:
    int("abc")
except ValueError as e:
    raise RuntimeError("Не удалось преобразовать данные.") from e

Собственные исключения

В Python можно создавать собственные классы исключений. Для этого нужно создать класс, наследующийся от Exception (или другого подходящего исключения).

Зачем создавать собственные исключения?

  • Делают код более читаемым — понятно, что именно пошло не так.
  • Позволяют разделить логику обработки разных типов ошибок.
  • Удобны при написании библиотек и модулей.

Базовый вариант — минимальный класс исключения

class NegativeValueError(Exception):
    pass

def square_root(n):
    if n < 0:
        raise NegativeValueError("Нельзя извлечь корень из отрицательного числа.")
    return n ** 0.5

try:
    print(square_root(-4))
except NegativeValueError as e:
    print(f"Ошибка: {e}")
# Ошибка: Нельзя извлечь корень из отрицательного числа.

Расширенный вариант — исключение с дополнительными данными

class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(
            f"Недостаточно средств. Баланс: {balance}, требуется: {amount}."
        )

class BankAccount:
    def __init__(self, balance):
        self.balance = balance

    def withdraw(self, amount):
        if amount > self.balance:
            raise InsufficientFundsError(self.balance, amount)
        self.balance -= amount
        return amount

account = BankAccount(100)

try:
    account.withdraw(200)
except InsufficientFundsError as e:
    print(f"Ошибка: {e}")
    print(f"Текущий баланс: {e.balance}")
# Ошибка: Недостаточно средств. Баланс: 100, требуется: 200.
# Текущий баланс: 100

Иерархия собственных исключений

В больших проектах удобно создавать базовое исключение приложения, от которого наследуются все остальные. Это позволяет ловить все ошибки приложения одним except:

class AppError(Exception):
    """Базовое исключение приложения."""
    pass

class DatabaseError(AppError):
    """Ошибки при работе с базой данных."""
    pass

class NetworkError(AppError):
    """Ошибки сети."""
    pass

class TimeoutError(NetworkError):
    """Превышено время ожидания."""
    pass

try:
    raise TimeoutError("Сервер не отвечает.")
except AppError as e:
    print(f"Ошибка приложения: {e}")
    print(f"Тип: {type(e).__name__}")
# Ошибка приложения: Сервер не отвечает.
# Тип: TimeoutError

Рекомендации по именованию

Имена собственных исключений принято заканчивать на Error или Exception, чтобы сразу было понятно, что это класс исключения. Например: ValidationError, AuthError, ParseException.
