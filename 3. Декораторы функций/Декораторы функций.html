Функции как объекты первого класса

В Python функции являются объектами первого класса (first-class objects). Это означает, что функцию можно присваивать переменной, передавать как аргумент другой функции, возвращать из функции и хранить в структурах данных — точно так же, как любое другое значение.

Что значит «объект первого класса»?

  • Функцию можно присвоить переменной.
  • Функцию можно передать как аргумент другой функции.
  • Функцию можно вернуть из функции.
  • Функцию можно хранить в списке, словаре и других структурах.

Присваивание функции переменной

Функция — это объект

def greet(name):
    return f"Привет, {name}!"

say_hello = greet

print(greet("Алиса"))     # Привет, Алиса!
print(say_hello("Боб"))   # Привет, Боб!

print(type(greet))        # <class 'function'>
print(greet.__name__)     # greet
print(id(greet) == id(say_hello))  # True — тот же объект

Передача функции как аргумента

Функции, принимающие другие функции в качестве аргументов, называются функциями высшего порядка.

def apply(func, value):
    return func(value)

def double(x):
    return x * 2

def square(x):
    return x ** 2

print(apply(double, 5))   # 10
print(apply(square, 5))   # 25

print(apply(str.upper, "hello"))  # HELLO
print(apply(len, [1, 2, 3, 4]))   # 4

Возврат функции из функции

Функция, возвращающая другую функцию, называется фабрикой функций. Именно на этом принципе основаны декораторы.

Фабрика функций-умножителей

def make_multiplier(factor):
    def multiplier(x):
        return x * factor
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))   # 10
print(triple(5))   # 15
print(double(triple(4)))  # 24

Хранение функций в структурах данных

import math

operations = {
    "sin": math.sin,
    "cos": math.cos,
    "sqrt": math.sqrt,
    "abs": abs,
}

for name, func in operations.items():
    print(f"{name}(4) = {func(4):.4f}")
# sin(4) = -0.7568
# cos(4) = -0.6536
# sqrt(4) = 2.0000
# abs(4) = 4.0000

Функции — это объекты с атрибутами

У каждой функции есть встроенные атрибуты: __name__, __doc__, __module__, __annotations__, __defaults__ и др.

def add(a: int, b: int = 0) -> int:
    """Складывает два числа."""
    return a + b

print(add.__name__)         # add
print(add.__doc__)          # Складывает два числа.
print(add.__annotations__)  # {'a': <class 'int'>, 'b': <class 'int'>, 'return': <class 'int'>}
print(add.__defaults__)     # (0,)

Замыкания

Замыкание (closure) — это вложенная функция, которая «запоминает» переменные из охватывающей области видимости даже после того, как внешняя функция завершила свою работу. Замыкания являются фундаментом для декораторов.

Три условия замыкания

  • Должна существовать вложенная функция.
  • Вложенная функция должна ссылаться на переменную из охватывающей области.
  • Внешняя функция должна возвращать вложенную функцию.

Простое замыкание — счётчик

def make_counter():
    count = 0

    def counter():
        nonlocal count
        count += 1
        return count

    return counter

counter1 = make_counter()
counter2 = make_counter()

print(counter1())  # 1
print(counter1())  # 2
print(counter1())  # 3
print(counter2())  # 1 — независимый счётчик
print(counter2())  # 2

Свободные переменные и ячейки

Переменные из охватывающей области, на которые ссылается замыкание, называются свободными переменными. Python хранит их в специальных объектах — ячейках (cell).

def make_adder(n):
    def adder(x):
        return x + n   # n — свободная переменная
    return adder

add5 = make_adder(5)
add10 = make_adder(10)

print(add5(3))    # 8
print(add10(3))   # 13

print(add5.__closure__)           # (<cell at 0x...>,)
print(add5.__closure__[0].cell_contents)  # 5

nonlocal — изменение переменной замыкания

По умолчанию вложенная функция может только читать переменную из внешней области. Чтобы изменить её, необходимо объявить переменную через nonlocal.

Аккумулятор с nonlocal

def make_accumulator(initial=0):
    total = initial

    def add(value):
        nonlocal total
        total += value
        return total

    return add

acc = make_accumulator(100)
print(acc(10))   # 110
print(acc(20))   # 130
print(acc(-5))   # 125

Замыкание с несколькими вложенными функциями

def make_power_functions():
    results = []

    for exp in range(1, 4):
        def power(x, e=exp):
            return x ** e
        results.append(power)

    return results

linear, quadratic, cubic = make_power_functions()

print(linear(3))     # 3
print(quadratic(3))  # 9
print(cubic(3))      # 27

Классическая ловушка с замыканиями в цикле

Если не использовать параметр по умолчанию (e=exp), все функции захватят одну и ту же ячейку, и после цикла она будет содержать последнее значение переменной.

funcs_wrong = [lambda x: x ** i for i in range(1, 4)]
print(funcs_wrong[0](3))  # 27 — неожиданно! i = 3 для всех

funcs_right = [lambda x, e=i: x ** e for i in range(1, 4)]
print(funcs_right[0](3))  # 3   — правильно

Написание декоратора с нуля

Декоратор — это функция, которая принимает другую функцию, добавляет к ней дополнительное поведение и возвращает новую (или ту же) функцию. Декораторы позволяют модифицировать функцию без изменения её исходного кода.

def decorator(func):
    def wrapper():
        # действия до вызова
        result = func()
        # действия после вызова
        return result
    return wrapper

Шаг за шагом: создаём первый декоратор

Декоратор логирования без синтаксиса @

def log_call(func):
    def wrapper():
        print(f"→ Вызов функции '{func.__name__}'")
        result = func()
        print(f"← Функция '{func.__name__}' завершена")
        return result
    return wrapper

def say_hello():
    print("Привет!")

say_hello = log_call(say_hello)
say_hello()
# → Вызов функции 'say_hello'
# Привет!
# ← Функция 'say_hello' завершена

Синтаксис @ — синтаксический сахар

Запись @decorator над определением функции — это краткая форма func = decorator(func). Оба варианта полностью эквивалентны.

def log_call(func):
    def wrapper():
        print(f"→ Вызов функции '{func.__name__}'")
        result = func()
        print(f"← Функция '{func.__name__}' завершена")
        return result
    return wrapper

@log_call
def say_hello():
    print("Привет!")

say_hello()
# → Вызов функции 'say_hello'
# Привет!
# ← Функция 'say_hello' завершена

Как Python выполняет @decorator

При загрузке модуля Python выполняет следующее преобразование:

@decorator
def func():
    ...

# Эквивалентно:
def func():
    ...
func = decorator(func)

Декоратор замера времени

Измерение времени выполнения

import time

def timer(func):
    def wrapper():
        start = time.perf_counter()
        result = func()
        elapsed = time.perf_counter() - start
        print(f"Время выполнения '{func.__name__}': {elapsed:.6f} сек")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(0.1)
    print("Готово!")

slow_function()
# Готово!
# Время выполнения 'slow_function': 0.100234 сек

Декораторы для функций с аргументами

Простой декоратор работает только для функций без аргументов. Чтобы декоратор был универсальным — подходил для любой функции — wrapper должен принимать *args и **kwargs и передавать их в оригинальную функцию.

def decorator(func):
    def wrapper(*args, **kwargs):
        # действия до
        result = func(*args, **kwargs)
        # действия после
        return result
    return wrapper

Универсальный декоратор логирования

def log(func):
    def wrapper(*args, **kwargs):
        args_repr = [repr(a) for a in args]
        kwargs_repr = [f"{k}={v!r}" for k, v in kwargs.items()]
        all_args = ", ".join(args_repr + kwargs_repr)
        print(f"Вызов: {func.__name__}({all_args})")
        result = func(*args, **kwargs)
        print(f"Вернул: {result!r}")
        return result
    return wrapper

@log
def add(a, b):
    return a + b

@log
def greet(name, greeting="Привет"):
    return f"{greeting}, {name}!"

add(3, 5)
# Вызов: add(3, 5)
# Вернул: 8

greet("Алиса", greeting="Здравствуй")
# Вызов: greet('Алиса', greeting='Здравствуй')
# Вернул: 'Здравствуй, Алиса!'

Декоратор проверки типов аргументов

def validate_positive(func):
    def wrapper(*args, **kwargs):
        for arg in args:
            if isinstance(arg, (int, float)) and arg < 0:
                raise ValueError(f"Аргументы должны быть положительными, получено: {arg}")
        return func(*args, **kwargs)
    return wrapper

@validate_positive
def sqrt(x):
    return x ** 0.5

print(sqrt(9))    # 3.0
print(sqrt(-4))   # ValueError: Аргументы должны быть положительными, получено: -4

Декоратор кэширования (простая версия)

def simple_cache(func):
    cache = {}

    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
            print(f"  Вычислено: {func.__name__}{args}")
        else:
            print(f"  Из кэша:   {func.__name__}{args}")
        return cache[args]

    return wrapper

@simple_cache
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(5))
# Вычислено: fibonacci(0,)
# Вычислено: fibonacci(1,)
# Вычислено: fibonacci(2,)
# ...
# 5

Стекирование декораторов

К одной функции можно применить несколько декораторов одновременно. Декораторы применяются снизу вверх (ближайший к функции — первым), а выполняются сверху вниз.

@decorator_a
@decorator_b
@decorator_c
def func():
    ...

# Эквивалентно:
func = decorator_a(decorator_b(decorator_c(func)))

Порядок применения декораторов

def bold(func):
    def wrapper(*args, **kwargs):
        return f"<b>{func(*args, **kwargs)}</b>"
    return wrapper

def italic(func):
    def wrapper(*args, **kwargs):
        return f"<i>{func(*args, **kwargs)}</i>"
    return wrapper

def underline(func):
    def wrapper(*args, **kwargs):
        return f"<u>{func(*args, **kwargs)}</u>"
    return wrapper

@bold
@italic
@underline
def greet(name):
    return f"Привет, {name}!"

print(greet("Алиса"))
# <b><i><u>Привет, Алиса!</u></i></b>

Стекирование практических декораторов

import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"Время: {time.perf_counter() - start:.4f} сек")
        return result
    return wrapper

def log(func):
    def wrapper(*args, **kwargs):
        print(f"Вызов: {func.__name__}")
        result = func(*args, **kwargs)
        print(f"Готово: {func.__name__}")
        return result
    return wrapper

@timer
@log
def process(n):
    return sum(range(n))

result = process(1_000_000)
# Вызов: process
# Готово: process
# Время: 0.0312 сек

Порядок имеет значение

Изменение порядка декораторов изменяет поведение. В примере выше @timer измеряет время всего вызова включая логи @log, потому что применяется снаружи. Поменяйте их местами — и @timer будет измерять только само тело функции.

functools.wraps

При обёртывании функции в декоратор создаётся новая функция wrapper. Это приводит к потере метаданных оригинальной функции: имени, строки документации, аннотаций типов и других атрибутов.

Проблема: метаданные теряются

def decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@decorator
def greet(name: str) -> str:
    """Возвращает приветствие."""
    return f"Привет, {name}!"

print(greet.__name__)         # wrapper  ← должно быть 'greet'
print(greet.__doc__)          # None     ← потеряли документацию
print(greet.__annotations__)  # {}       ← потеряли аннотации

Решение: @functools.wraps

Декоратор @functools.wraps(func) копирует все важные атрибуты из оригинальной функции в wrapper: __name__, __qualname__, __doc__, __dict__, __annotations__, __module__, __wrapped__.

@functools.wraps(wrapped, assigned=WRAPPER_ASSIGNMENTS, updated=WRAPPER_UPDATES)
Декоратор для использования внутри декораторов. Копирует метаданные функции wrapped на функцию-обёртку. Также добавляет атрибут __wrapped__, указывающий на оригинальную функцию.

Правильное использование @wraps

from functools import wraps

def decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@decorator
def greet(name: str) -> str:
    """Возвращает приветствие."""
    return f"Привет, {name}!"

print(greet.__name__)         # greet   ✅
print(greet.__doc__)          # Возвращает приветствие.  ✅
print(greet.__annotations__)  # {'name': <class 'str'>, 'return': <class 'str'>}  ✅
print(greet.__wrapped__)      # <function greet at 0x...>  — ссылка на оригинал

Полноценный декоратор с @wraps

from functools import wraps
import time

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"[{func.__name__}] выполнено за {elapsed:.6f} сек")
        return result
    return wrapper

@timer
def compute_sum(n: int) -> int:
    """Возвращает сумму чисел от 0 до n."""
    return sum(range(n))

result = compute_sum(1_000_000)
print(result)
# [compute_sum] выполнено за 0.031842 сек
# 499999500000

print(compute_sum.__name__)  # compute_sum  ✅
print(compute_sum.__doc__)   # Возвращает сумму чисел от 0 до n.  ✅

__wrapped__ для интроспекции

Атрибут __wrapped__ позволяет добраться до оригинальной функции, минуя все декораторы. Это полезно в тестах и инструментах интроспекции.

@timer
@timer
def my_func():
    pass

print(my_func.__wrapped__)            # функция, обёрнутая первым @timer
print(my_func.__wrapped__.__wrapped__)  # оригинальная my_func

Декораторы с аргументами

Иногда декоратору нужно передать параметры. Для этого создаётся фабрика декораторов — функция, принимающая аргументы и возвращающая сам декоратор. Появляется дополнительный уровень вложенности.

def decorator_factory(arg1, arg2):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            # использует arg1, arg2
            return func(*args, **kwargs)
        return wrapper
    return decorator

@decorator_factory(arg1, arg2)
def func():
    ...

Как Python выполняет @decorator_factory(args)

Запись @decorator_factory(arg) выполняется в два шага:

@decorator_factory(arg)
def func():
    ...

# Эквивалентно:
decorator = decorator_factory(arg)  # шаг 1: вызов фабрики
func = decorator(func)              # шаг 2: применение декоратора

Декоратор repeat — повторяет вызов N раз

from functools import wraps

def repeat(times):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            result = None
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def greet(name):
    print(f"Привет, {name}!")

greet("Алиса")
# Привет, Алиса!
# Привет, Алиса!
# Привет, Алиса!

Декоратор retry — повторяет при исключении

from functools import wraps
import time

def retry(times=3, delay=1.0, exceptions=(Exception,)):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            last_error = None
            for attempt in range(1, times + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    last_error = e
                    print(f"Попытка {attempt}/{times} не удалась: {e}")
                    if attempt < times:
                        time.sleep(delay)
            raise last_error
        return wrapper
    return decorator

import random

@retry(times=3, delay=0.1)
def unstable_request():
    if random.random() < 0.7:
        raise ConnectionError("Нет соединения")
    return "Успех!"

print(unstable_request())

Декоратор validate_type — проверка типов аргументов

from functools import wraps

def validate_types(**type_map):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            import inspect
            sig = inspect.signature(func)
            bound = sig.bind(*args, **kwargs)
            bound.apply_defaults()
            for param_name, expected_type in type_map.items():
                value = bound.arguments.get(param_name)
                if value is not None and not isinstance(value, expected_type):
                    raise TypeError(
                        f"Параметр '{param_name}' должен быть {expected_type.__name__}, "
                        f"получено {type(value).__name__}"
                    )
            return func(*args, **kwargs)
        return wrapper
    return decorator

@validate_types(name=str, age=int)
def register(name, age):
    print(f"Зарегистрирован: {name}, {age} лет")

register("Алиса", 25)    # Зарегистрирован: Алиса, 25 лет
register("Боб", "20")    # TypeError: Параметр 'age' должен быть int, получено str

Декоратор, работающий и с аргументами, и без

Иногда удобно, чтобы декоратор мог использоваться как с аргументами (@decorator(arg)), так и без (@decorator). Это достигается через проверку типа первого аргумента.

from functools import wraps, partial

def repeat(_func=None, *, times=1):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper

    if _func is not None:
        return decorator(_func)
    return decorator

@repeat
def say_once():
    print("Раз!")

@repeat(times=3)
def say_three():
    print("Три!")

say_once()   # Раз!
say_three()  # Три! Три! Три!

Три уровня вложенности декоратора с аргументами

Уровень Функция Принимает Возвращает
1 (фабрика) decorator_factory Аргументы декоратора Декоратор
2 (декоратор) decorator Функцию func wrapper
3 (обёртка) wrapper *args, **kwargs Результат func

Практические примеры

Декораторы широко применяются в реальных проектах. Рассмотрим наиболее полезные и часто встречающиеся паттерны.

Декоратор авторизации

Проверка прав доступа

from functools import wraps

CURRENT_USER = {"name": "Алиса", "role": "admin"}

def require_role(role):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            if CURRENT_USER.get("role") != role:
                raise PermissionError(
                    f"Доступ запрещён. Требуется роль: {role}, "
                    f"у пользователя: {CURRENT_USER.get('role')}"
                )
            return func(*args, **kwargs)
        return wrapper
    return decorator

@require_role("admin")
def delete_user(user_id):
    print(f"Пользователь {user_id} удалён")

@require_role("superadmin")
def shutdown_server():
    print("Сервер остановлен")

delete_user(42)    # Пользователь 42 удалён
shutdown_server()  # PermissionError: Доступ запрещён. Требуется роль: superadmin

Декоратор замера времени с порогом

Предупреждение о медленных вызовах

from functools import wraps
import time

def slow_warning(threshold=1.0):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            start = time.perf_counter()
            result = func(*args, **kwargs)
            elapsed = time.perf_counter() - start
            if elapsed > threshold:
                print(f"⚠ '{func.__name__}' выполнялась {elapsed:.2f} сек (порог: {threshold} сек)")
            return result
        return wrapper
    return decorator

@slow_warning(threshold=0.5)
def fast():
    time.sleep(0.1)

@slow_warning(threshold=0.5)
def slow():
    time.sleep(1.0)

fast()  # (без предупреждения)
slow()  # ⚠ 'slow' выполнялась 1.00 сек (порог: 0.5 сек)

Декоратор для синглтона

Паттерн Singleton через декоратор класса

def singleton(cls):
    instances = {}

    def get_instance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]

    return get_instance

@singleton
class DatabaseConnection:
    def __init__(self, host):
        self.host = host
        print(f"Подключение к {host}")

db1 = DatabaseConnection("localhost")  # Подключение к localhost
db2 = DatabaseConnection("localhost")  # (без вывода — тот же объект)

print(db1 is db2)  # True

Декоратор deprecation warning

Пометить функцию как устаревшую

from functools import wraps
import warnings

def deprecated(message=""):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            warn_msg = f"'{func.__name__}' устарела и будет удалена в следующих версиях."
            if message:
                warn_msg += f" {message}"
            warnings.warn(warn_msg, DeprecationWarning, stacklevel=2)
            return func(*args, **kwargs)
        return wrapper
    return decorator

@deprecated("Используйте новую функцию compute() вместо неё.")
def old_compute(x):
    return x * 2

result = old_compute(5)
# DeprecationWarning: 'old_compute' устарела ... Используйте новую функцию compute() вместо неё.
print(result)  # 10
